Задача: реализовать патч 1.1 по следующему ТЗ.

=== ПАТЧ 1.1 ===

Цель патча:
Усилить ощущение игры про вампира, сделать охоту осмысленной, бой вариативным, а прогрессию интереснее через способности.

1. Навигация и экраны
- Нижнее меню пока оставить как есть по общей структуре.
- В Hub вынести отдельные кнопки:
  - Прокачка
  - Способности
  - Инвентарь
- Прокачка = покупка способностей.
- Способности = просмотр уже купленных/доступных у игрока.
- Social:
  - Лидеры
  - Рефералы
  - всё остальное пометить как "Скоро".

2. Hunt / охота
- Hunt больше не должен восприниматься как прямой запуск боя.
- Hunt = запуск поиска события.
- Сначала игрок выбирает локацию.
- В патче 1.1 полноценная стартовая локация одна: Грот.
- Лес пока не реализовывать как полноценную локацию.

3. Энка / ночь / рассвет
- Энка = ресурс ночной охоты.
- Максимум энки = 100.
- Один поиск события = 10 энки.
- При 0 энки охота завершается.
- Игрок возвращается в Hub.
- Запускается КД 4 часа.
- После КД энка восстанавливается до 100.

4. Кристалл ночи
- Разовый расходник на текущую сессию охоты.
- Даёт +40 энки.
- Используется только когда энка = 0.
- В этот момент показать игроку вопрос, хочет ли он активировать кристалл.
- После использования кристалла охота продолжается.
- После следующего завершения охоты всё равно наступает обычный КД.
- Это не бесконечное продление ночи.

5. События локации Грот
Реализовать события:
- Бой:
  - крыса
  - паук
- Сундук:
  - зелье лечения
- Жертва:
  - волк в паутине
  - сразу переход к экрану финального выбора без полноценного боя
- Пустое событие:
  - "Ничего не произошло"

6. Боевой экран
Перестроить боевой экран так, чтобы критично важные элементы влезали без неудобного скролла.
Порядок блоков:
- карточка врага
- статы врага: HP / атака / защита
- текстовый лог / обработка события
- кнопки боя:
  - Атака
  - Навык
  - Убежать
- если есть фамильяр — он отображается чуть выше игрока
- блок игрока:
  - HP
  - атака
  - защита
  - голод
- Кнопку "Поглотить" убрать из основного боя.

7. Финал боя
- При последнем смертельном ударе враг не умирает мгновенно.
- Вместо этого открывается финальный выбор:
  - Добить
  - Выпить досуха
  - Гипноз

8. Награды финального выбора
Добить:
- +50% опыта от базовой награды
- HP не восстанавливает
- голод не восстанавливает

Выпить досуха:
- обычный опыт
- восстановить HP, равный значению противника
- восстановить голод до 100

Гипноз:
- опыт как у "выпить"
- шанс успеха 10%
- при успехе враг становится фамильяром
- при провале просто выдать опыт, без штрафа и без контратаки

9. Голод
- Добавить голод в блок статов игрока.
- Навыки тратят голод.
- При низком голоде урон игрока немного увеличивается.
- При низком голоде кнопка "Убежать" недоступна.
- Голод восстанавливается до 100 при действии "Выпить досуха".

10. Фамильяры
- Фамильяр — не отдельная коллекционная система, а боевой расходный ресурс.
- Максимум 1 фамильяр в бою.
- Максимум 1 фамильяр вне боя.
- Если фамильяр выжил, он переносится в следующий бой.
- Если погиб, исчезает.
- Фамильяр появляется в бою автоматически.
- Враг атакует фамильяра первым, пока он жив.
- Фамильяр добавляет свой отдельный урон в боевой лог.
Пример:
- Вампир нанёс 14 урона
- Крыса укусила на 3 урона

11. Призыв летучих мышей
- Это боевой навык-призыв, а не обычный фамильяр.
- Навык доступен только если у игрока нет активного фамильяра.
- Это временный защитный ресурс.
- Базовая концепция:
  - 1 HP
  - высокий шанс уклонения
  - принимают удар вместо игрока
- Числа можно реализовать близко к этой логике и при необходимости пометить на дальнейший баланс-тест.

12. Донатные фамильяры
- Донатные фамильяры не являются отдельными боевыми юнитами.
- Это пассивные усиления.
- Выбираются в Hub.
- Меняются вне охоты.
- Дают бонусы, но не должны превращаться в pay-to-win.

13. Прокачка и способности
- Прокачка строится через карточки способностей.
- Опыт используется как валюта для покупки способностей.
- Есть пассивные и активные способности.
- Пассивных должно быть больше.
- Именно набор пассивных карт формирует билд.
- Ветки способностей:
  - Охота
  - Выживание
  - Господство

14. Базовые статы
- Старые базовые статы пока не удалять.
- Они остаются фундаментом.
- Карточки способностей работают поверх них.

15. Скины
- Скины должны отображаться:
  - в карточке игрока в Hub
  - в бою вместо простой текстовой надписи "вампир"
- Пока реализовать только портрет.
- Без рамок, анимаций и косметических усложнений.

16. Ограничения патча
Сейчас НЕ делать:
- полноценный сложный лут
- глубокую переработку экономики
- ручное управление фамильярами
- много новых локаций
- анимированные рамки
- новые соц-механики кроме лидеров и рефералов

17. Требования к работе
- Реализуй патч поэтапно.
- Если удобнее, сначала перестрой UX состояний и навигацию, потом бой, потом способности, потом Social/Shop-встраивание.
- После реализации обязательно проверь ключевые сценарии:
  1. вход в Hub
  2. переход в Hunt
  3. выбор локации Грот
  4. трата энки при поиске
  5. бой
  6. финальный выбор
  7. поведение голода
  8. работа фамильяра
  9. возврат в Hub после 0 энки
  10. предложение активировать Кристалл ночи
  11. отображение портрета игрока
  12. открытие экранов Прокачка / Способности / Инвентарь / Social

Формат работы:
- Не вываливай сразу огромную простыню рассуждений.
- Сначала коротко опиши план реализации.
- Затем выполняй изменения по шагам.
- Используй skill по необходимости.
- После выполнения дай:

  - что осталось спорным
  - какие места требуют ручного теста
  - какие файлы были изменены
