# Coding Rules

## Обязательно
- Делать минимально достаточные изменения под задачу.
- Сохранять существующий стиль кода и соглашения проекта.
- Добавлять/обновлять проверки для изменённой логики.
- Обрабатывать ошибки явно и предсказуемо.
- Убирать мёртвый код и неиспользуемые импорты в затронутых файлах.

## Запрещено
- Массовые форматирующие правки в несвязанных файлах.
- Изменение публичных контрактов без фиксации последствий.
- Добавление зависимостей без явного обоснования.
- Скрытие ошибок через пустые `catch` и немые fallback-и.

## Желательно
- Делить сложную логику на небольшие функции.
- Писать самодокументируемый код и короткие полезные комментарии.
- Сначала решать корректность, потом микрооптимизации.
