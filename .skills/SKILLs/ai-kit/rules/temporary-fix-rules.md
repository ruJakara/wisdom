# Temporary Fix Rules

## Обязательно
- Каждое временное решение помечать явным маркером (`TEMP_FIX`, `WORKAROUND`).
- Указывать причину, область действия и условие удаления временного фикса.
- Локализовать временный фикс в минимальной зоне кода.
- Добавлять задачу на удаление/замену временного решения.

## Запрещено
- Выдавать временный фикс за финальную архитектуру.
- Распространять workaround на новые модули без крайней необходимости.
- Оставлять временные решения без документации и владельца.
- Использовать временный фикс вместо устранения первопричины, если причина уже известна.

## Желательно
- Ставить срок пересмотра временного решения.
- Добавлять проверку, что workaround не ломает соседние сценарии.
- В итоговом отчёте отдельно перечислять все оставшиеся временные фиксы.
