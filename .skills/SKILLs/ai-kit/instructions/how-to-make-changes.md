# How to Make Changes

## Цель
Вносить правки безопасно и предсказуемо.

## Шаги
1. Выбери минимальный набор файлов для изменения.
2. Убедись, что изменения соответствуют архитектурным границам.
3. Сначала внеси функциональные изменения, затем косметику.
4. Делай изменения малыми порциями и проверяй после каждой.
5. Сохраняй обратную совместимость, если иное не согласовано.
6. Обнови тесты/проверки и документацию рядом с изменением.
7. Удали временные костыли или пометь их с планом удаления.

## Быстрые проверки
- Нет затронутых несвязанных модулей.
- Нет молчаливых изменений публичных контрактов.
- Нет дублирования логики без причины.
