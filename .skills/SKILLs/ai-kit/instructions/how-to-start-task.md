# How to Start a Task

## Цель
Быстро и корректно начать задачу, не делая преждевременных правок.

## Шаги
1. Зафиксируй задачу в 1-2 предложениях: что нужно получить и зачем.
2. Определи тип работы: feature, bugfix, refactor, архитектура, документация.
3. Выпиши критерии готовности: что считается "сделано".
4. Определи ограничения: сроки, совместимость, запреты, доступные инструменты.
5. Составь короткий план из 3-7 шагов.
6. Проверь, есть ли риски или неоднозначности; если есть, зафиксируй допущения.
7. Только после этого переходи к чтению и изменениям.

## Контрольный список старта
- Понятен ожидаемый результат.
- Понятны границы задачи.
- Есть проверяемые критерии завершения.
- План зафиксирован до правок.
