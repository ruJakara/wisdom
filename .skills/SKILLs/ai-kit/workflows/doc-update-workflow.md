# Doc Update Workflow

## Цель
Поддерживать документацию синхронной с реальным состоянием проекта.

## Шаги
1. Определить, какие изменения в проекте требуют обновления docs.
2. Найти документы-владельцы этих областей.
3. Внести точечные правки без переписывания лишнего.
4. Проверить согласованность терминов и ссылок.
5. Удалить или пометить устаревшие части.
6. Проверить, что документ реально помогает выполнить задачу.
7. Зафиксировать, что именно обновлено.

## Гейт перед завершением
- Документы отражают фактическое поведение.
- Нет конфликтов между связанными файлами.
- Нет "мертвых" инструкций.
