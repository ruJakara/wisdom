# Integration Workflow

## Цель
Надёжно связать несколько систем в один рабочий контур.

## Шаги
1. Составить карту интеграций: системы, владельцы, протоколы, контракты.
2. Согласовать форматы данных, версии API и схему ошибок.
3. Подготовить окружения и конфиги без утечки секретов.
4. Сшивать интеграции по одной связке, а не всё сразу.
5. Для каждой связки проверить happy-path и failure-path.
6. Добавить наблюдаемость: структурные логи, id трассировки, метрики ошибок.
7. Прогнать критичный end-to-end сценарий целиком.
8. Зафиксировать итоговые контракты и ограничения в docs.

## Гейт перед завершением
- Сквозной сценарий проходит стабильно.
- Ошибки интеграции диагностируются без ручной "магии".
- Документация и конфиги соответствуют фактической реализации.
