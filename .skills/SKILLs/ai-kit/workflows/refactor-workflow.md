# Refactor Workflow

## Цель
Улучшить структуру кода без изменения поведения.

## Шаги
1. Определить проблемный участок и целевое улучшение.
2. Зафиксировать неизменяемое внешнее поведение.
3. Разбить работу на небольшие шаги.
4. После каждого шага проверять эквивалентность поведения.
5. Остановиться при достижении практической цели.
6. Обновить архитектурные заметки при изменении границ модулей.
7. Подготовить отчёт о структурных изменениях.

## Гейт перед завершением
- Функциональное поведение сохранено.
- Код заметно проще и чище.
- Нет лишних побочных правок.
