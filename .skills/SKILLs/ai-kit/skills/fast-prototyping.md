# Fast Prototyping

## Title
Fast Prototyping

## Purpose
Собирать рабочие прототипы быстро, сохраняя минимальную инженерную дисциплину для дальнейшего развития.

## Scope
- Быстрое создание MVP-функционала.
- Сборка демо-потоков для проверки гипотез.
- Подготовка прототипа к безопасному переходу в production-план.

## Inputs
- Гипотеза или бизнес-идея.
- Ограничения по времени и стеку.
- Минимальные критерии успеха демо.

## Outputs
- Рабочий прототип с основным пользовательским потоком.
- Явные ограничения и технические компромиссы.
- Список шагов для перевода в production-качество.

## Responsibilities
- Фокусироваться на одном главном сценарии.
- Держать код простым и очевидным.
- Маркировать временные решения и быстрые упрощения.
- Сразу фиксировать, что требует усиления позже.

## Constraints
- Не превращать прототип в бесконтрольный "сарай".
- Не усложнять архитектуру до проверки гипотезы.
- Не скрывать временные решения как финальные.

## What to avoid
- Оптимизация до подтверждения ценности.
- Слишком широкая функциональность в первом проходе.
- Зависимости и инфраструктура, не нужные для проверки гипотезы.

## Typical tasks
- Поднять MVP веб-прототипа за 1-3 итерации.
- Быстро собрать вертикальный slice (UI -> API -> storage).
- Подготовить демо для валидации идеи с пользователями.

## Quality checklist
- Основной сценарий проходит от начала до конца.
- Ограничения прототипа явно задокументированы.
- Временные решения промаркированы.
- Есть понятный план следующего шага.
