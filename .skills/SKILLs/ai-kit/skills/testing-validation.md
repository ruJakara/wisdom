# Testing & Validation

## Title
Testing and Validation

## Purpose
Подтверждать корректность изменений и снижать риск регрессий.

## Scope
- Проверка функциональных и граничных сценариев.
- Сопоставление результата с критериями задачи.
- Подготовка минимального набора воспроизводимых проверок.

## Inputs
- Описание изменений и ожидаемого поведения.
- Доступные тестовые инструменты проекта.
- Критические пользовательские сценарии.

## Outputs
- Список выполненных проверок и их результат.
- Обнаруженные риски/ограничения проверки.
- Рекомендации по дополнительным тестам при необходимости.

## Responsibilities
- Выбрать разумный минимум проверок под риск задачи.
- Покрыть happy-path и ключевые edge-cases.
- Ясно отделить "проверено" от "не проверено".

## Constraints
- Не заявлять успех без фактической проверки.
- Не игнорировать ошибки тестов/линтеров без причины.
- Не подменять проверку субъективным впечатлением.

## What to avoid
- Только ручная проверка при высоком риске регрессии.
- Проверки "для галочки" без связи с задачей.
- Отсутствие фиксации шагов воспроизведения.

## Typical tasks
- Прогнать unit/integration/e2e релевантного контура.
- Добавить тест на найденный баг.
- Проверить миграции, конфиги и обратную совместимость.

## Quality checklist
- Критические сценарии проверены.
- Результаты проверок зафиксированы.
- Известные непокрытые зоны явно указаны.
- Рекомендации по усилению тестирования конкретны.
