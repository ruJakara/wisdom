# Integration Stitching

## Title
Integration Stitching

## Purpose
Сшивать несколько систем в единый рабочий поток без потери устойчивости и наблюдаемости.

## Scope
- Интеграция API, ботов, БД, очередей, админок, платежей, лидербордов и внешних сервисов.
- Согласование контрактов данных и ошибок между компонентами.
- Введение безопасных точек синхронизации и fallback-обработки.

## Inputs
- Контракты интерфейсов и форматы данных.
- Конфиги окружений и секреты (или их плейсхолдеры).
- Набор целевых end-to-end сценариев.

## Outputs
- Сквозной рабочий интеграционный поток.
- Таблица контрактов/endpoint-ов/событий между системами.
- Проверки критичных интеграционных сценариев.

## Responsibilities
- Явно фиксировать ownership для каждой интеграционной точки.
- Обрабатывать ошибки и таймауты по обе стороны стыка.
- Сохранять трассируемость: логи, correlation id, понятные сообщения.
- Обновлять интеграционную документацию при каждом изменении контракта.

## Constraints
- Не менять сразу все интеграции одним большим коммитом.
- Не вводить неявные преобразования данных без документации.
- Не хранить секреты в коде и репозитории.

## What to avoid
- Хрупкие связки "работает только в одном окружении".
- Склеивание модулей через копипасту DTO/моделей.
- Игнорирование идемпотентности и повторной доставки событий.

## Typical tasks
- Сшить backend API и Telegram/Discord бота.
- Подключить платежный шлюз к учёту в БД и админке.
- Подружить игровой сервер с таблицей лидеров и профилями.

## Quality checklist
- Все контракты синхронизированы и проверены.
- Критичные e2e-сценарии проходят.
- Ошибки интеграций диагностируются по логам.
- Изменения конфигов и зависимостей документированы.
