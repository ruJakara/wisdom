# Documentation Writing

## Title
Documentation Writing

## Purpose
Делать документацию, которая ускоряет работу и снижает количество повторных вопросов.

## Scope
- Технические инструкции, карты проекта, workflow, правила.
- Обновления README, API-описаний и operational notes.

## Inputs
- Реальные изменения в коде/процессах.
- Текущие документы проекта.
- Целевая аудитория документа (разработчик, AI-агент, поддержка).

## Outputs
- Краткий и точный документ с практическими шагами.
- Обновлённые ссылки между связанными файлами.
- Явные ограничения и предположения.

## Responsibilities
- Писать только проверяемые и применимые инструкции.
- Поддерживать согласованность терминов и структуры.
- Удалять или помечать устаревшие разделы.

## Constraints
- Не документировать несуществующую функциональность.
- Не копировать длинные куски без пользы.
- Не оставлять двусмысленных формулировок.

## What to avoid
- Общие фразы без шагов и критериев.
- Документацию "про всё" вместо конкретной задачи.
- Противоречивые инструкции в разных файлах.

## Typical tasks
- Обновить README после изменения запуска.
- Добавить workflow для нового типа задач.
- Описать архитектурное решение и последствия.

## Quality checklist
- Документ решает конкретную практическую задачу.
- Текст короткий, структурный, без воды.
- Есть привязка к реальным файлам/шагам/командам.
- Нет конфликтов с соседними документами.
