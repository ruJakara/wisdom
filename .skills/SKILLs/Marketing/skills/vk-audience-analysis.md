# vk-audience-analysis

## Title
VK Audience Analysis

## Purpose
Построить рабочую картину целевой аудитории и её психологии до написания поста.

## Scope
- Анализ идеи поста, описания услуги, кейса, комментариев и вопросов.
- Выделение сегментов ЦА, барьеров, триггеров доверия, контент-гипотез.
- Подготовка структурированного вывода для writer-этапа.

## Inputs
- Идея поста или задача пользователя.
- Контекст бренда (RAG/проектные материалы).
- При наличии: посты конкурентов, отзывы, комментарии.

## Outputs
- Сегменты аудитории.
- Боли, желания, страхи, сомнения.
- Что цепляет и что мешает покупке/доверию.
- Набор гипотез для контента.

## Responsibilities
- Отделять факты от выводов и гипотез.
- Давать выводы, пригодные для прямой передачи в post writer.
- Явно указывать неопределённость при нехватке данных.

## Constraints
- Нельзя писать готовый продающий пост вместо анализа.
- Нельзя выдавать предположения за проверенные факты.
- Нельзя использовать манипулятивные или агрессивные интерпретации.

## What to avoid
- Общие фразы без связи с входными данными.
- Категоричные выводы при слабом контексте.
- Дублирование writer-задач (формулировка готового текста поста).

## Typical tasks
- Разбор идеи поста перед публикацией.
- Анализ поста конкурента для адаптации подхода.
- Выявление причин слабой реакции аудитории на прошлый контент.

## Quality checklist
- Есть разделение факт/вывод/гипотеза.
- Выводы можно использовать как вход для writer без доп. расшифровки.
- Указаны конкретные боли/мотивы, а не абстрактные характеристики.
- Нет готовых рекламных обещаний и "продажи в лоб".
