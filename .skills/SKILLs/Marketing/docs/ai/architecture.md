# Architecture

## Общая схема проекта

Текущая архитектура документарная и построена как линейный конвейер:

`Контекст проекта + запрос -> Audience Analysis -> Post Writer -> Итоговый пост VK`.

## Основные модули

1. `Content Director` (`skills/vk-content-director.md`)
2. `Audience Analysis` (`skills/vk-audience-analysis.md`)
3. `Post Writer` (`skills/vk-post-writer.md`)

## Ответственность модулей

- `Content Director`: определяет порядок работы и следит, чтобы этап анализа не пропускался.
- `Audience Analysis`: извлекает сегменты ЦА, боли, мотивы, барьеры и контент-гипотезы.
- `Post Writer`: формирует финальный пост с хуком и мягким CTA на основе анализа.

## Связи между модулями

- `Director` запускает `Audience Analysis`.
- Вывод `Audience Analysis` является входом для `Post Writer`.
- `Post Writer` не должен заменять собой аналитический модуль.

## Хранение состояния

Постоянного runtime-состояния нет.

- состояние сессии хранится в контексте диалога;
- долговременные правила зафиксированы в markdown-файлах проекта.

## Внешние интеграции

- Платформа публикации: VK (на уровне целевого формата текста).
- Контекст бренда: внешний RAG/материалы пользователя (в коде проекта не реализован как отдельный модуль).

## Зоны риска

- Пропуск аналитического этапа перед генерацией поста.
- Подмена фактов гипотезами без маркировки.
- Потеря авторского стиля при написании текста.
- Рассинхрон между старыми `SKILL *.md` и новыми файлами в `skills/`.

## Точки расширения

- Добавить отдельный skill для контент-плана (серии постов, а не единичного).
- Добавить skill для разбора постов конкурентов как самостоятельный модуль.
- Добавить папку с проверенными примерами входов/выходов для калибровки качества.
