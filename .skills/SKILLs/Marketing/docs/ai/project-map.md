# Project Map

## Краткое назначение проекта

Проект `Marketing` хранит AI-навыки и правила для создания контента VK в двух этапах:

1. анализ аудитории;
2. написание поста на основе анализа.

## Ключевые папки

- `docs/ai/` - системная документация по архитектуре, правилам и процессу.
- `skills/` - актуальные skill-файлы с единым форматом.

## Где находится основная логика

Основная логика находится в markdown-инструкциях:

- `skills/vk-content-director.md` - оркестрация этапов.
- `skills/vk-audience-analysis.md` - аналитический этап.
- `skills/vk-post-writer.md` - генерация поста.

## Где данные

Явных хранилищ данных в репозитории нет.

- входные данные приходят из запроса пользователя;
- контекст бренда ожидается через внешний RAG/проектные материалы (вне этого каталога).

## Где ассеты

Выделенной папки ассетов в проекте нет.
Идеи визуалов формируются текстом внутри контентных задач.

## Где инструменты

- `Taskfile.yml` - базовые проверки структуры и skill-шаблонов.

## Где входные точки

- Исторические входные файлы: `SKILL analyst VK-.md`, `SKILL post_writer.md`, `SKILL VK director.md`.
- Актуальные входные файлы для развития: каталог `skills/`.

## Куда вносить изменения по типу задачи

- Нужно поправить порядок этапов: `skills/vk-content-director.md` + `docs/ai/architecture.md`.
- Нужно улучшить анализ аудитории: `skills/vk-audience-analysis.md`.
- Нужно изменить формат/тон постов: `skills/vk-post-writer.md`.
- Нужно обновить общие стандарты работы: `docs/ai/rules.md`.
- Нужно закрепить критерии приёмки: `docs/ai/done-criteria.md`.
