# Rules

## Обязательно

- Сохранять 2-этапную модель: `анализ аудитории -> создание поста`.
- Для каждого skill в `skills/` держать единый формат секций (Title, Purpose, Scope и т.д.).
- При изменении поведения навыка обновлять связанные файлы в `docs/ai/`.
- Явно отделять факты, выводы и гипотезы в аналитических инструкциях.
- Вносить точечные изменения вместо полной переписи файлов без необходимости.
- Использовать понятные и предметные имена файлов (`vk-...`, `...-analysis`, `...-writer`).

## Запрещено

- Пропускать этап анализа перед генерацией поста, если задача требует финальный текст.
- Добавлять в навыки несуществующие интеграции, API или автоматизации.
- Вводить новые зависимости/инструменты без реальной необходимости для проекта.
- Оставлять временные костыли без пометки и без плана удаления.
- Засорять skills общими фразами без конкретных действий.
- Использовать манипулятивные или недостоверные обещания в шаблонах контента.

## Желательно

- Держать документы короткими, но операционными.
- Добавлять примеры входа/выхода, если это снижает неоднозначность.
- Отдельно фиксировать, где предположение, а где факт проекта.
- Синхронизировать старые и новые skill-файлы, избегая дублирующих правил.
- Использовать комментарии в markdown только там, где они объясняют нетривиальное решение.
