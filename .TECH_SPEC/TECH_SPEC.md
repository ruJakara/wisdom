## 1. Общие сведения о проекте
**Название проекта:** Night Hunger: Vampire Evo  
**Платформа:** Telegram Mini Apps (TMA)  
**Жанр:** Arcade Simulator / Survival / RPG Lite  
**Целевая аудитория:** Пользователи Telegram 16-35 лет, любители казуальных игр и кликеров.  
**Цель:** Создание вирусной игры с простым геймплеем, глубокой системой прогрессии и социальной интеграцией.

## 2. Концепция игры
Игрок управляет вампиром в ночном городе. Основная цель — выжить, насытиться кровью и эволюционировать из слабого новичка в древнего владыку тьмы.

### 2.1. Core Loop (Основной цикл)
1. **Поиск:** Игрок перемещается по карте (или выбирает локации).
2. **Взаимодействие:** Встреча с NPC (Люди, Охотники, Другие вампиры).
3. **Действие:** 
   - Если враг слабый → **Атака** (Поедание).
   - Если враг сильный → **Побег** (Уклонение).
   - Если NPC мирный → **Кормление** (Питье крови).
4. **Награда:** Получение опыта (XP) и Крови (Валюта).
5. **Прогрессия:** Прокачка характеристик и покупка улучшений.
6. **Повтор:** Возврат к шагу 1 с новыми силами.

## 3. Игровая механика

### 3.1. Характеристики персонажа
| Параметр | Описание | Влияние |
| :--- | :--- | :--- |
| **Сила (Strength)** | Мощность укуса | Урон по слабым врагам, скорость поедания. |
| **Ловкость (Agility)** | Скорость движения | Шанс убежать от сильных врагов, скорость перемещения. |
| **Кровь (HP/Energy)** | Здоровье/Ресурс | Тратится на действия, восполняется питьем. Если 0 → Проигрыш/Респаун. |
| **Уровень (Level)** | Общий прогресс | Открывает новые локации и скины. |

### 3.2. Типы NPC
1. **Мирные жители:** Не опасны. Дают мало XP, много Крови. Механика: *Пить кровь*.
2. **Слабые враги (Бродяги):** Низкий урон. Дают среднее XP. Механика: *Убить/Съесть*.
3. **Сильные враги (Охотники):** Высокий урон. Механика: *Убегать (QTE или мини-игра)*. При успехе → бонус. При провале → потеря Крови.
4. **Боссы (Вампиры-старейшины):** Появляются периодически. Требуют подготовки.

### 3.3. Система боя/взаимодействия
- **Режим карты:** 2D вид сверху или упрощенный интерфейс карточек (для MVP).
- **Бегство:** Мини-игра "нажимай кнопку быстро" или проверка стата Ловкости против стата Врага.
- **Атака:** Проверка стата Силы. Если Сила > Защиты врага → Победа.

## 4. Технический стек

### 4.1. Frontend (Клиентская часть)
- **Фреймворк:** React.js или Vue.js (для реактивного интерфейса).
- **Стилизация:** Tailwind CSS (быстрая верстка).
- **Telegram Integration:** `@twa-dev/sdk` или официальный `telegram-web-app.js`.
- **Анимации:** Framer Motion или CSS Animations.
- **Графика:** Canvas API или Pixi.js (если планируется активное движение на карте).

### 4.2. Backend (Серверная часть)
- **Язык:** Node.js (NestJS/Express) или Python (FastAPI/Aiogram).
- **База данных:** PostgreSQL (хранение пользователей, инвентаря) + Redis (кэширование сессий, лидерборды).
- **Авторизация:** Через `initData` Telegram (валидация на бэкенде).

### 4.3. Инфраструктура
- **Хостинг:** VPS (Ubuntu) или Serverless (Vercel/AWS Lambda).
- **Бот:** Python (Aiogram) или Node.js (Telegraf) для обработки команд `/start`, уведомлений.

## 5. Интерфейс (UI/UX)
**Стиль:** Темный, неоновый, готический (цвета: черный, кроваво-красный, фиолетовый).

### 5.1. Основные экраны
1. **Главный экран (Hub):** 
   - Статус персонажа (HP, Уровень).
   - Кнопка "На охоту".
   - Кнопка "Прокачка".
2. **Экран охоты:** 
   - Визуализация встречи (карточка врага).
   - Кнопки действий (Атака, Пить, Бежать).
   - Таймер действия.
3. **Магазин/Прокачка:** 
   - Список улучшений характеристик.
   - Магазин скинов (внешний вид вампира).
4. **Лидерборд:** 
   - Топ игроков по уровню или съеденным жертвам.

## 6. Монетизация
1. **Telegram Stars:** Покупка валюты внутри игры для ускорения прокачки.
2. **Реклама (Ads):** 
   - Просмотр рекламы для воскрешения (если HP = 0).
   - Просмотр рекламы для получения x2 Крови на 30 минут.
3. **Premium доступ:** Ежемесячная подписка (уникальный скин, отсутствие рекламы, бонус к опыту).

## 7. Социальные механики (Telegram Native)
- **Реферальная система:** "Призови друга-вампира" → получи бонус крови.
- **Шеринг:** Кнопка "Поделиться результатом охоты" в чаты.
- **Уведомления:** Бот присылает сообщение: "Твой вампир проголодался! Зайди в игру".

## 9. Риски и ограничения
1. **Производительность:** Игра должна быстро грузиться на мобильных устройствах (вес бандла < 2MB).
2. **Политика Telegram:** Строгие правила regarding азартных игр и крипто-токенов (если планируется).
3. **Безопасность:** Вся логика начисления валюты должна быть на сервере (Anti-cheat).

## 10. Требования к коду
- Чистая архитектура (Clean Architecture).
- Комментарии к сложным функциям.
- Использование TypeScript (рекомендуется).
